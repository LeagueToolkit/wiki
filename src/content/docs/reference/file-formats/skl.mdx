title: 'SKL (Skeletons)'
description: Technical reference for the SKL skeleton format.
---

SKL files define the skeletal hierarchy used to animate League of Legends models. A skeleton consists of a tree of joints (bones) with transforms, parent relationships, and influence mappings that connect the skeleton to a [skinned mesh](/reference/file-formats/skn/).

## Overview

Every animated model in League has an associated skeleton that defines its bone structure. The skeleton provides:

- **Joint hierarchy** - Parent-child relationships between bones
- **Local transforms** - Position, rotation, and scale of each joint relative to its parent
- **Inverse bind transforms** - Transforms used to convert vertices from model space to bone space for skinning
- **Influence list** - Maps mesh vertex bone indices to skeleton joints

SKL files are found inside [WAD archives](/reference/file-formats/wad/) at paths like `DATA/Characters/{name}/Skins/Base/{name}.skl`.

## Use Cases

- **Custom animations** - Understanding the bone hierarchy to create compatible animations
- **Model rigging** - Matching skeleton structure when creating custom skins
- **Skeleton inspection** - Analyzing bone hierarchies, joint counts, and influence mappings

## File Structure

All multi-byte values are **little-endian**.

The SKL format has two distinct branches: the **modern format** (used by current game versions) and the **legacy format** (found in older game data).

## Modern Format

### Detection

The modern format is identified by a format token at offset 4:

| Offset | Size | Type | Field        | Description                              |
|--------|------|------|--------------|------------------------------------------|
| 0x00   | 4    | u32  | file_size    | Total file size in bytes                 |
| 0x04   | 4    | u32  | format_token | `0x22FD4FC3` (identifies modern format)  |

### Header (64 bytes)

| Offset | Size | Type | Field              | Description                                    |
|--------|------|------|--------------------|------------------------------------------------|
| 0x00   | 4    | u32  | file_size          | Total file size in bytes                       |
| 0x04   | 4    | u32  | format_token       | `0x22FD4FC3`                                   |
| 0x08   | 4    | u32  | version            | Format version (must be 0)                     |
| 0x0C   | 2    | u16  | flags              | Format flags                                   |
| 0x0E   | 2    | u16  | joint_count        | Number of joints                               |
| 0x10   | 4    | u32  | influence_count    | Number of influence mappings                   |
| 0x14   | 4    | i32  | joints_offset      | Offset to joint data array                     |
| 0x18   | 4    | i32  | joint_indices_offset| Offset to joint index-hash pairs              |
| 0x1C   | 4    | i32  | influences_offset  | Offset to influence array                      |
| 0x20   | 4    | i32  | name_offset        | Offset to skeleton name string                 |
| 0x24   | 4    | i32  | asset_name_offset  | Offset to asset name string                    |
| 0x28   | 4    | i32  | bone_names_offset  | Offset to bone name string table               |
| 0x2C   | 20   | i32[5] | ext_offsets      | Reserved extension offsets                     |

### Joint Data (100 bytes each)

Each joint entry in the joints array:

| Offset | Size | Type  | Field              | Description                                   |
|--------|------|-------|--------------------|-----------------------------------------------|
| 0x00   | 2    | u16   | flags              | Joint flags                                   |
| 0x02   | 2    | u16   | id                 | Joint ID                                      |
| 0x04   | 2    | i16   | parent_id          | Parent joint ID (-1 for root)                 |
| 0x06   | 2    | u16   | padding            | Padding                                       |
| 0x08   | 4    | u32   | name_hash          | [ELF hash](/reference/hashing/algorithms/#elf-hash) of the joint name (lowercased) |
| 0x0C   | 4    | f32   | radius             | Joint radius (for collision/selection)         |
| 0x10   | 12   | Vec3  | local_position     | Local translation (x, y, z)                   |
| 0x1C   | 12   | Vec3  | local_scale        | Local scale (x, y, z)                         |
| 0x28   | 16   | Quat  | local_rotation     | Local rotation quaternion (x, y, z, w)        |

The inverse bind transform is decomposed into translation (Vec3), scale (Vec3), and rotation (Quat) components rather than stored as a full 4×4 matrix:

| Offset | Size | Type  | Field                    | Description                              |
|--------|------|-------|--------------------------|------------------------------------------|
| 0x38   | 12   | Vec3  | inv_bind_translation     | Inverse bind translation (x, y, z)       |
| 0x44   | 12   | Vec3  | inv_bind_scale           | Inverse bind scale (x, y, z)             |
| 0x50   | 16   | Quat  | inv_bind_rotation        | Inverse bind rotation (x, y, z, w)       |
| 0x60   | 4    | i32   | name_offset              | Relative offset to null-terminated name string |

### Joint Index-Hash Pairs

At `joint_indices_offset`, an array of entries mapping IDs to hashes:

| Size | Type | Field    | Description              |
|------|------|----------|--------------------------|
| 2    | i16  | joint_id | Joint ID                 |
| 2    | u16  | padding  | Padding                  |
| 4    | u32  | hash     | ELF hash of joint name (lowercased) |

### Influence List

At `influences_offset`, an array of `influence_count` entries:

| Size | Type | Field      | Description                                        |
|------|------|------------|----------------------------------------------------|
| 2    | i16  | joint_index| Index into the joints array for this influence     |

The influence list maps [SKN](/reference/file-formats/skn/) vertex bone indices to skeleton joints. When a vertex references bone index `N`, it maps to `influences[N]` which gives the actual joint index in the skeleton.

## Legacy Format

### Detection

The legacy format is identified by an 8-byte ASCII magic:

| Offset | Size | Type   | Field | Description              |
|--------|------|--------|-------|--------------------------|
| 0x00   | 8    | char[] | magic | `r3d2sklt` (ASCII)       |

### Legacy Header

| Offset | Size | Type | Field        | Description                    |
|--------|------|------|--------------|--------------------------------|
| 0x00   | 8    | char[]| magic       | `r3d2sklt`                     |
| 0x08   | 4    | u32  | version      | Legacy version (1 or 2)        |
| 0x0C   | 4    | u32  | joint_count  | Number of joints               |

### Legacy Joint Data

Each joint in the legacy format:

| Size | Type   | Field          | Description                              |
|------|--------|----------------|------------------------------------------|
| 32   | char[] | name           | Null-padded ASCII joint name             |
| 4    | i32    | parent_id      | Parent joint index (-1 for root)         |
| 4    | f32    | radius         | Joint radius                             |
| 48   | f32[12]| global_matrix  | 3×4 global transform matrix              |

### Version Differences (Legacy)

| Version | Influences                                           |
|---------|------------------------------------------------------|
| 1       | Influences generated automatically from joint indices |
| 2       | Explicit influence array follows joint data           |

## Version Summary

| Format | Identifier        | Version | Joint Size | Features                        |
|--------|--------------------|---------|------------|---------------------------------|
| Modern | `0x22FD4FC3`       | 0       | 100 bytes  | Full transforms, name hashes, extensions |
| Legacy | `r3d2sklt`         | 1       | ~88 bytes  | Auto-generated influences       |
| Legacy | `r3d2sklt`         | 2       | ~88 bytes  | Explicit influences             |
