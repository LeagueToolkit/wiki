A [mod project](/making-mods/mod-projects/) is a directory with a
`mod.config.json` and one or more content layers. You author it, validate it,
then pack it into a `.modpkg` that anyone can install.

There are two ways to do this and they operate on the same project format:

- **[The Creator Workshop](/making-mods/workshop/)** - the project editor built
  into LTK Manager. Recommended if you are starting out.
- **[The `league-mod` CLI](/making-mods/command-line/)** - the same workflow
  from a terminal, and the engine the Workshop is built on. Useful for
  scripting, CI, and batch work.

Projects are interchangeable between them. Nothing you build in one is locked
away from the other.

## Before you start

You will get much further if you know what the game files you are editing
actually are. Two pages worth reading first:

- **[How Modding Works](/start-here/how-modding-works/)** - how your files end up
  in the running game.
- **[WAD Archives](/reference/file-formats/wad/)** - the container almost
  everything lives in.

The rest of the [format reference](/reference/file-formats/wad/) covers meshes,
skeletons, animations, textures, and property bins as you need them.
